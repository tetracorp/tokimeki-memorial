### Notes and analysis

This file listing is correct for both the original release and limited edition
releases (SLPS-00065 and SLPS-00064), which are identical except for the
serial number in the disc metadata.

The creation on the disc metadata is `1995-08-31 12:00:00.00 36`, with 36
referring to 36 15-minute increments ahead of UTC, or +09:00 (Japan time). I
suspect this means the file dates on the disc are Japanese local time, given
that the file date for the root dir is 11:00. However, I can't be certain that
the dates shouldn't be adjusted +09:00 for Japanese time.

Either way, the staff appear to have worked long or irregular hours. There are
file dates at weekends.

The earliest date is 30 March 1995, six and a half months before launch. With
some games, you might see files dating back to the game's development.
Most of these early files are music files and game intro and outro work, with
the intro animation `openxa.str` dated 15 June 1995.

It shouldn't be inferred that they started by making the intro/outro first.
Rather, these files simply were simply finalized at this point and did not
change later. A lot of files are closely sequentially dated in alphabetical
order, which is a sign that they were copied in a batches from another source,
which then may have updated them to reflect the copy dates rather than original
creation dates.

Observe how long it took to copy files in 1995. In `cdrom/prolodir/`, 21 files
for a total of 1.4MB copied in 22 seconds. `cdrom/xa_g/kagami.xa`, 32MB, took
almost 14 minutes to copy. Certainly, most of the voices were finalized by 13
August, someone working a Sunday no less, though for some reason `katagiri.xa`
was updated on the 15th.

The last copy batches in the week before the CD is mastered are `exedir` on 28
Aug, and the main game EXE on 31 Aug right before the image for the gold master
is finalized. A possible explanation is that a "final" build took place at one
point which was copied over on 28 Aug, followed by a quick bugfix on the 31st
right before mastering. The release of further patches suggests that they were
not done bugfixing on the 31st, but had to produce a working build to meet a
month-end deadline.

In terms of development timeline, I my guess would be that the developers began
work on the PlayStation port of this game after the original PC Engine CD
release which, according to Wikipedia, came out on 27 May 1994. The PlayStation
did not come out in Japan until 3 December 1994, but according to
[this article](https://www.retroreversing.com/official-playstation-1-software-development-kit-(psyq)/),
and [this article](https://www.retroreversing.com/official-playStation-devkit),
the Psy-Q development software used by _Tokimeki Memorial_ and countless other
games was already in widespread use by November 1994, and in development as
early as mid-1993. Konami likely saw the opportunity to port their game to the
PlayStation within the platform's first year.

### Analysis of file changes in subsequent releases

A second revision was released with the volume set ID `SLPS00065VER125`. This is
the same product serial as the standard release, but the version has updated
from 110 to 125. Its burn date is 17 November 1995 at 00:00 Japan time (a false
time).

    1995-06-15 13:52:30 /openxa.str
    1995-06-17 14:29:56 /konamic3.str
    1995-06-21 05:11:14 /system.cnf
    1995-08-18 09:50:28 /cdrom/
    1995-08-31 10:03:26 /psx.exe
    1995-11-17 00:00:00 /

The filesystem under `/cdrom/` can't be read normally (perhaps an anti-piracy or
anti-ripping measure), so this is the limit of the file listing. The file
timestamps haven't changed, but the root directory has updated to match the new
burn date.

However, the main executable `psx.exe` _has_ changed, despite the identical
time. The filesize is identical (690176 bytes), but the md5sum has changed from
`136d832a30948b8533a786c5ea7e1dd1` (in the initial v110 release) to
`b8450a8720d65517415e964f58994fe3`.

The intro video files (the konami logo video `konamic3.str` and game intro
animation `openxa.str` have also been modified in this version. The filesizes
and disc sector locations are unchanged. The only difference I have noticed is
that the individual files play in jPSXdec without an error about missing raw
headers. Future builds use these versions of the files.

More research has to be done to discover what changes have been made in this new
build. Additionally, it's possible that other changes were made, but examining
changes is impeded by the lack of a standard file listing (remember that this
game accesses files by disc sector number rather than ISO9660 filename).

A fourth revision (I have yet to see a third revision) exists with the volume
set ID `SLPS00065VER143`, updating us from v125 to v143. Its disc burn date is
31 March 1996.

    1995-06-15 13:52:30 /openxa.str
    1995-06-17 14:29:56 /konamic3.str
    1995-06-21 05:11:14 /system.cnf
    1995-08-18 09:50:28 /cdrom/
    1995-08-31 10:03:26 /psx.exe
    1996-03-31 00:00:00 /

Again, the disc write time is updated but not the datetime on visible files. The
main executable appears to have changed again: it now has an md5sum of
`a86925fed256c674339c5988c06fd0e3` and a filesize of 667648 bytes, unless some
read error has occurred.

Finally, there's the _Playstation the Best_ release of the game. It's dated 25
December 1997, and the preparer ID has changed from "K.FUKUHARA" to
"K_FUKUHARA", suggestion perhaps that Fukuhara re-entered their name
differently after all this time, perhaps now using a new PC to re-master the
release.

The volume ID of this version is no longer the version number, reading VX009J2.
The main EXE filename has changed from `psx.exe` to a new standard-style
`SLPM_86.053`. 667648 bytes with an md5sum of
`aa6eb063d80f0ff563f926456de734cb`. All files have now updated to the new date,
with time set to zero.

    1997-12-25 00:00:00 /
    1997-12-25 00:00:00 /cdrom/
    1997-12-25 00:00:00 /konamic3.str
    1997-12-25 00:00:00 /openxa.str
    1997-12-25 00:00:00 /slpm_86.053
    1997-12-25 00:00:00 /system.cnf

The Japanese PlayStation website for the Playstation the Best version
(serial [SLPM-86053](https://www.jp.playstation.com/software/title/slpm86053.html))
lists Christmas 1997 as the release date of this version, so it's probably not
the real build date. Fukuhara worked hard on this game but probably not on
Christmas day.

There's also a 2003 PSone re-release which I haven't personally seen,
[SLPM-87300](https://www.jp.playstation.com/software/title/slpm87300.html). I
don't know if any changes were made to the code or it's unmodified from
SLPM-86053. Likewise with the 2009
[digital download](https://www.jp.playstation.com/software/title/jp0101npjj00303_000000000000000001.html)
release.

The exact differences between the various versions of the game require more
research. There are definitely code differences between the various executables,
and there may be differences between other game files which are harder to read
because they are not listed in the normal filesystem.
